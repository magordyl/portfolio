Planner V1 · Design Explorer v2 · 2026-04-12

Calm by default. Bright when it matters.

A rebuild against a tighter brief: calmness and preparedness, not alarm. Three grounded primaries — pine, harbor, slate — each paired with a warm celebration accent that only appears at the dopamine moments (streaks, completion milestones, weekly wins). The frame is still locked: warm-paper canvas, Fraunces × Inter × JetBrains Mono, dense mobile-first, snappy 150–200ms motion. The canvas is slightly softer than v1 so it sits better against cooler primaries.

What changed from v1 → dropped the warm-red / bramble candidates (read as warning tones). Introduced a dual-accent system: primary handles everyday surfaces (checkmarks, FAB, active tab, selected weekday, primary buttons, SMS user replies); celebration handles gamification moments only (streak chips, review wins, milestone callouts). Softened canvas saturation for better pairing with cool primaries.

The dual-accent rulebook

Primary accent — the quiet workhorse. Used on every everyday surface:

  • Task checkmarks (done state)
  • Floating action button (new task)
  • Active tab bar item + indicator dot
  • Selected weekday in weekly review
  • Primary button fill (Mark done, Next, Save)
  • SMS thread user reply accent rail
  • Overdue task time (signalled by weight + colour, never red)

Celebration accent — the bright sparing dopamine hit. Only appears when the app is recognising a win:

  • Streak chips (header banner + per-habit count badges)
  • Weekly review opening callout (last week's completion rate)
  • Task detail streak callout (days running + personal best)
  • Done-state micro-animation (flash of celebration on the check, 180ms, fades to primary)
  • Monthly milestone badges (when they ship)

Rule of thumb: if you'd describe the moment as "the app noticing you did something meaningful," it uses celebration. Everything else uses primary. The distinction matters — if celebration leaks into everyday UI, the dopamine stops working.
