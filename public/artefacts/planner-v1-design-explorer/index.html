Type system

Fraunces · Inter · JetBrains Mono

Fraunces at high opsz for display, dropped to 14-opsz italic for editorial body (SMS thread messages, lede copy). Inter for interface body and interactive text. JetBrains Mono at 10–12px uppercase for metadata, timestamps, and command tokens — the "engineer's fingerprint" on an otherwise editorial surface.

What's locked, what's still up for grabs

Locked across all candidates:

  • Canvas — warm monochrome tonal ramp, 36°→30° hue drift
  • Type — Fraunces display (variable opsz), Inter body, JetBrains Mono metadata
  • Density — dense mobile-first rhythm (14px body, 10–12px meta, 8–12px row padding)
  • Motion — snappy 150–200ms, opacity + transform only, no bounce
  • SMS as typographic nod — Fraunces italic for incoming, mono for replies; no iMessage mimicry

The only thing varying above is --accent — a single HSL triplet used for completed checkmarks, overdue state, FAB, primary button, active tab, and selected weekday. Everything else inherits the locked frame unchanged.

Next step — pick A, B, or C (or redirect with what's missing), then this direction feeds the Phase 3 design.tokens.ts rewrite as part of R1. No Phase 5 second round — one decisive pass per the /new-feature skill.
