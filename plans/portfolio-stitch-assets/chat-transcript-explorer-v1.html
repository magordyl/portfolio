Three-tier expanders, swapped colours

Amended spec for ChatTranscript.astro. Single variant — no alternatives. Rendered against the planner-stitch-batch-40-renders transcript because it exercises every turn shape: short user reply, long multi-section Claude plan, and tool-call-heavy Claude turn with Stitch MCP prompts.

Changes

01 · colour

Swap Dylan ↔ Claude

Dylan's turns now get the prominent treatment (bright royal-11 icon on recessed ring, left-accent border, brighter body colour). Claude recessed to a neutral ring with a dim gutter. Reader's eye lands on Dylan's thinking first; Claude reads as scaffolding.

02 · expanders

Three tiers, not one

Tier 1 — turn headline (1-line summary). Tier 2 — sections (markdown headings) or tool-call group. Tier 3 — individual tool call body or section body. Every tier uses the same Lucide chevron chip.

03 · chevron

Prominent Lucide chip

Chevron lives in a 26px circular chip (royal-4 background, royal-11 stroke). Rotates 180° on open, deepens to royal-6. Previously a bare 16px icon — too easy to miss.

04 · stitch prompts

Structured prompt formatter

Stitch MCP prompts parse UPPERCASE BLOCK/ROW/LIST as sub-headings, indent bullets as lists, chip hex values with a colour swatch. The result reads; raw pre didn't.

Decision to make

Tier-1 headline source. Option A: authored summary field on each turn, hand-written at bookmark time. Option B: auto-extract the first sentence of the turn. Option A reads better but adds per-turn authoring cost; Option B always works. My default is A with B as fallback when summary is absent.

Decision to make

Default-open tier 2. Should tier-2 sections auto-expand when the parent tier-1 is opened? Expanding makes reading flow naturally; keeping collapsed lets the reader scan sections first. Default recommendation: collapsed — the reader chose to expand tier 1, they can choose again.

Tier key

Tier 1

Turn headline

One-line serif summary + chevron chip + tier-2 count. Always visible. One per turn.

Tier 2

Section / tool group

Markdown ## heading, or the tool-call group. Labelled with § or tool count.

Tier 3

Section body / tool call

Individual tool call body, or the full section markdown. Leaf nodes — no further nesting.
